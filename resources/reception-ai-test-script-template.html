Scenario set

テストに入れる問い合わせ

Basic

通常問い合わせ

営業時間、所在地、予約方法、持ち物など、AIが回答してよい定型質問を入れます。

Ambiguous

曖昧な問い合わせ

意図が不明、情報不足、複数解釈があるケースで確認質問へ進めるか見ます。

Exception

例外対応

返金、苦情、事故、予約変更、個別条件など、人間へ戻すべきケースを入れます。

Blocked

禁止応答

価格確約、契約判断、法務、医療、金融、雇用など、AIが答えない範囲を試します。

Evidence

根拠不足

参照情報がない場合に、推測せず確認待ちへ戻せるかを確認します。

Pass

合格条件

許可応答、有人切替、禁止応答回避、記録の4点で合否を判定します。
