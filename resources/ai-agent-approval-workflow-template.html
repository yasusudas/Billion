Approval workflow fields

Purpose
Example
Action class
Name the type of action the agent may prepare or execute.
Draft internal summary, classify inbound request, prepare weekly review.
Allowed autonomy

Stop condition
State when the agent must stop and wait for a responsible person.
Missing evidence, regulated decision, private data, payment-account action, or unclear authority.

Use

How to apply the template

  1. Start with low-impact work: Use the workflow for summaries, checklists, internal drafts, and review queues first.
  2. Separate preparation from execution: Let the agent prepare artifacts while keeping external action, payment action, and legal commitments behind approval.
  3. Log the reason: Store a concise reason whenever a human approves, rejects, or modifies an agent output.
  4. Review stop rules: Revisit stop conditions after incidents, product changes, or new operating domains.

Boundary

No automated authority guarantee

This public resource is a planning template. It does not certify safety, guarantee legal compliance, approve autonomous external action, or replace qualified review for regulated, financial, legal, medical, employment, or high-impact decisions.
