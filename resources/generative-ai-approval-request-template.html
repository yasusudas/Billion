Request sections

稟議に入れる項目

Purpose

導入目的

時間削減、品質平準化、抜け漏れ防止など、業務上の目的を1つずつ分けます。

Scope

対象業務

下書き、分類、要約、チェックなど、AIに任せる範囲を業務名で書きます。

Data

入力データ境界

公開情報、社内一般情報、顧客情報、機密情報を分け、投入禁止範囲を明記します。

Review

人間確認点

外部送信、公開、契約、請求、削除、権限変更の前に確認者を置きます。

Fallback

停止条件

誤回答、根拠不明、機密入力、例外対応、苦情など、AI利用を止める条件を決めます。

Evidence

運用証跡

入力要約、出力要約、修正理由、承認者、レビュー日を残す方法を書きます。
