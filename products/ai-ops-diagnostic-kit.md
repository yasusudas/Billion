# AI Ops Diagnostic Kit

Billion AI Operations Studio

## Purpose

Use this kit to review internal AI workflows before expanding agent, LLM, or automation usage. It is designed for operational planning and does not provide legal, security, financial, medical, hiring, or regulatory compliance advice.

## Files

- `ai-ops-control-matrix.csv`
- `ai-ops-risk-checklist.csv`
- `ai-ops-10-day-plan.csv`

## How to Use

1. List the AI workflows currently used or planned.
2. Complete the control matrix for each workflow.
3. Run the risk checklist before any rollout or tool-permission change.
4. Use the 10-day plan to assign owners, review gaps, and decide whether a custom implementation project is justified.

## Acceptance Criteria

- Every workflow has a named owner.
- Data classes and tool permissions are documented.
- A human approval point exists before external impact.
- Stop conditions are written in plain language.
- Evidence notes avoid personal data, private logs, and credentials.
